🌊 Understanding Bubble Sort

Ever watched bubbles rise to the top in water? That's how Bubble Sort works — the largest values "bubble" up with each round. Let's break it down step-by-step, like you're seeing it in the visualizer.

🔍 Step-by-Step Breakdown

  1. Start at the beginning of the array
  2. Compare two adjacent values
  3. If the left one is larger 👉 swap them
  4. Move one step right ➡️ and repeat
  5. Once a full pass is done, last item is sorted
  6. Repeat until the array is completely sorted ✅

🎓 Walkthrough Example

Let’s sort this: [34, 12, 25]

🔁 Compare 34 & 12 → Swap ➡️ [12, 34, 25]

🔁 Compare 34 & 25 → Swap ➡️ [12, 25, 34] ✅

🍾 34 is now sorted at the end! Repeat for the rest...

🏁 Final array: [12, 25, 34]

⏱️ Time Complexity of Bubble Sort

Scenario Time Complexity Description
Best Case O(n) When the array is already sorted. Only one pass is needed with no swaps.
Average Case O(n²) On average, it compares and possibly swaps each pair multiple times.
Worst Case O(n²) When the array is in completely reverse order. Max comparisons and swaps needed.

🧠 Space Complexity: O(1)

💡 Pro Tip

Each round locks one more value in place. So if you see green boxes at the end, those are done and won't move again!
